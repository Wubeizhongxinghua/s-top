# Code of Conduct

## Our standard

This project expects respectful, constructive, and professional behavior in issues, pull requests, and discussions.

Examples of acceptable behavior:

- asking questions in good faith
- giving technical feedback respectfully
- focusing on reproducible facts and user impact
- being considerate when discussing cluster environments and institutional constraints

Examples of unacceptable behavior:

- personal attacks
- harassment or intimidation
- discriminatory language
- deliberately disruptive behavior
- sharing private or sensitive security details publicly without coordination

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, commits, or pull requests that violate this policy.

## Reporting

If you experience or witness unacceptable behavior, contact the maintainer through GitHub. If the report is security-sensitive, follow [SECURITY.md](SECURITY.md) instead of posting publicly.
